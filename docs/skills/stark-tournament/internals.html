Architecture Lens

Control Plane

Input normalization is explicit: inline prompt or YAML config, then defaults for strategy, competitors, judge, and timeout. The wrapper owns validation and setup visibility.

Execution Plane

The wrapper delegates real work to tournament.py, which runs competitors in parallel, evaluates results, and tolerates partial competitor failure.

Output Plane

Two render paths exist: direct human-readable passthrough or JSON parsing into a ranked score table. Both terminate by surfacing audit location and optional artifact retention.

Extension Points And Failure Containment

Weighted Winner Selection

Phase 3 sorts competitors by weighted average descending and decorates the top row with a winner marker. The rendered table is stable even when factor sets change because it derives columns from judged output.

9.0+ excellent 7.0+ good 5.0+ acceptable <5.0 poor

Strategy-Specific Inputs

semantic is the generic path. visual changes how outputs are judged. test introduces an additional hard dependency: --test-file must be present.

This is the cleanest extension seam for specialized tournament types because the wrapper already forwards flags transparently.

Primary Modification Levers

Most customization lives in four places: competitor ID registry, evaluation strategy enum, factor weight vocabulary, and judge model selection. The wrapper also exposes prompt templating through --variables.

Because all user flags pass through unchanged, new script capabilities can often be adopted with little or no wrapper churn.

Failure Isolation Policy

No prompt or config: ask the user. Bad config path or invalid YAML: fail early with a precise error. Single competitor failure or timeout: disqualify and continue. All competitors fail: hard error. Judge failure: fall back to the first valid competitor output and mark the audit with eval_failed: true.

Observability Surface

The skill follows the shared observability protocol for timings and checkpoints, then adds tournament-specific metrics: competitors dispatched, succeeded, failed, factor count, strategy, winner ID, winner score, and total duration.

This makes the skill easy to compare across runs even when prompts or model rosters vary.

Artifact Trail

The terminal output is not the source of truth. The durable record is the JSONL audit under ~/.claude/code-review/history/tournaments/. When --keep-all is enabled, the skill also points to the output directory holding all competitor artifacts for later inspection.
